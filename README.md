# FitWeaver — Garmin Workout Generator

Canonical repository: `https://github.com/AIminov/FitWeaver`

> 🇷🇺 [Русская версия](README.ru.md)

Generate structured Garmin workouts from plain-text training plans using a local LLM.
Deliver them to your watch via USB or directly to **Garmin Connect Calendar** — no cable needed.

---

## Pipeline

```
Plan text  →  LLM  →  YAML  →  direct build  ─┬─  .fit  →  USB  →  Watch
                                               └─  Garmin Connect Calendar  →  Watch sync
```

The full workflow requires no intermediate Python templates.
`--templates-only` and `--build-only` modes are retained as legacy/debug tools.

---

## Delivery Options

| Method | How | Best for |
|--------|-----|----------|
| **USB** | Copy `.fit` to `/GARMIN/New files` | Single workouts, offline |
| **Garmin Calendar** | `garmin-calendar` CLI command | Full plans, automatic scheduling |

## Usage Scenarios

### Scenario 1 — Local LLM

Fully local pipeline: plain-language plan → LLM converts to YAML → FIT files are built.

**Requirements:**
- [LM Studio](https://lmstudio.ai/) or [Ollama](https://ollama.com/) with a model loaded

**Steps:**

**1.** Install dependencies:
```bash
pip install -r requirements.txt
```

**2.** Write your training plan in free form and save to `Plan/`:
```
Plan/my_plan.md
```

Example content:
```
10-week running plan. 3 workouts per week.
Monday: easy run 40 min, HR 130–145
Wednesday: intervals 6×800m, HR 165–175, 2 min recovery
Saturday: long run 60 min, HR 125–140
```

**3.** Run LLM generation:

*LM Studio:*
```bash
python -m garmin_fit.llm.request_cli \
  --plan Plan/my_plan.md \
  --api openai \
  --url http://127.0.0.1:1234/v1 \
  --openai-mode completions \
  --timeout-sec 1800
```

*Ollama:*
```bash
python -m garmin_fit.llm.request_cli \
  --plan Plan/my_plan.md \
  --api ollama \
  --model llama3
```

> **Optional:** If your plan uses zone names ("Z2", "easy pace") instead of explicit bpm values,
> create `user_profile.yaml` so the LLM knows your personal zones:
> ```bash
> cp user_profile.yaml.example user_profile.yaml
> ```

If the workout count is not auto-detected, the script will ask:
```
Could not auto-detect workout count from plan structure.
How many workouts does the plan contain? (Enter to skip): 30
```

Or specify upfront:
```bash
python -m garmin_fit.llm.request_cli --plan Plan/my_plan.md --workouts 30 ...
```

**4.** Build FIT files:
```bash
python -m garmin_fit.cli run
```

**5.** Connect your Garmin watch and copy files from `Output_fit/`:

- Copy `.fit` files to the **`/GARMIN/New files`** folder on the watch
- The watch will process the files automatically — they will appear in **`/GARMIN/Workouts`**

On the watch:
select any running mode → **Training → Workouts** → choose the desired workout.

### Scenario 2 — Pre-made plan from Claude / ChatGPT

If you already have a YAML plan (generated by Claude, ChatGPT, or written manually) — no local LLM needed.

**Steps:**

**1.** Ask Claude or ChatGPT to generate YAML following the schema:

Prompt:
```
Generate a training plan in YAML format for the Garmin FIT Workout Generator project.
The schema is described here: https://github.com/AIminov/FitWeaver/blob/main/docs/YAML_GUIDE.md
```

**2.** Save the resulting YAML to `Plan/`:
```
Plan/my_plan.yaml
```

**3.** Install dependencies:
```bash
pip install -r requirements.txt
```

**4.** Optionally validate the plan before building:
```bash
python -m garmin_fit.cli validate-yaml --plan Plan/my_plan.yaml
```

The validator checks HR ranges, formats, name uniqueness, and repeat block logic.

**5.** Build FIT files:
```bash
python -m garmin_fit.cli run
```

Or directly specifying the file:
```bash
python -m garmin_fit.cli run --plan Plan/my_plan.yaml
```

**6.** Copy files from `Output_fit/` to your watch:

- Copy `.fit` files to the **`/GARMIN/New files`** folder on the watch
- The watch will process the files automatically — they will appear in **`/GARMIN/Workouts`**

On the watch:
select any running mode → **Training → Workouts** → choose the desired workout.

> **Tip:** If the YAML from Claude or ChatGPT fails validation — paste the errors back
> to the model and ask it to fix them. Usually one iteration is enough.

### Scenario 3 — Garmin Connect Calendar (no USB)

Upload the full plan directly to Garmin Connect. Workouts appear on your watch after the next sync — no cable required.

**Requirements:**
- A Garmin Connect account
- Workout filenames in `W{week}_{MM-DD}_{...}` format (e.g. `W11_03-14_Sat_Long_14km`) — date is extracted automatically

**Upload the full plan:**
```bash
python -m garmin_fit.cli garmin-calendar \
  --plan Plan/my_plan.yaml \
  --email your@email.com \
  --password yourpassword \
  --year 2026
```

**Preview without making any API calls:**
```bash
python -m garmin_fit.cli garmin-calendar --plan Plan/my_plan.yaml --dry-run
```

**Upload a specific date range:**
```bash
python -m garmin_fit.cli garmin-calendar \
  --plan Plan/my_plan.yaml \
  --email your@email.com \
  --password yourpassword \
  --year 2026 \
  --from-date 2026-06-01 \
  --to-date 2026-06-30
```

**Delete uploaded workouts from a date range:**
```bash
python -m garmin_fit.cli garmin-calendar-delete \
  --email your@email.com \
  --password yourpassword \
  --year 2026 \
  --from-date 2026-06-01 \
  --to-date 2026-06-30 \
  --dry-run

python -m garmin_fit.cli garmin-calendar-delete \
  --email your@email.com \
  --password yourpassword \
  --year 2026 \
  --from-date 2026-06-01 \
  --to-date 2026-06-30 \
  --confirm
```

**All options:**

| Flag | Default | Description |
|------|---------|-------------|
| `--plan` | auto-detect | Path to YAML plan |
| `--email` | `GARMIN_EMAIL` env | Garmin Connect email |
| `--password` | `GARMIN_PASSWORD` env | Garmin Connect password |
| `--token-dir` | `~/.garminconnect` | Token cache directory |
| `--year` | current/next | Override year for date extraction |
| `--dry-run` | off | Build payloads, make no API calls |
| `--no-schedule` | off | Upload without placing in calendar |
| `--skip-past` | off | Skip workouts whose date is before today |
| `--from-date` | none | Only upload on or after this date (YYYY-MM-DD) |
| `--to-date` | none | Only upload on or before this date (YYYY-MM-DD) |
| `--week-pause` | 3.0 s | Pause between calendar weeks (rate-limit protection) |

Credentials can be passed as flags or set via `GARMIN_EMAIL` / `GARMIN_PASSWORD` environment variables.
Tokens are cached after first login — subsequent runs skip authentication.

> **SBU drills:** each exercise gets its own repeat group with the drill name shown as a step note on the watch (e.g. "High Knees", "Bounds", "Accelerations").

## Quick Start

**1.** Install dependencies:
```bash
pip install -r requirements.txt
```

**2.** Place your plan in:
```
Plan/plan.md   or   Plan/plan.txt
```

**3.** Generate YAML via LLM (LM Studio):
```bash
python -m garmin_fit.llm.request_cli --api openai --url http://127.0.0.1:1234/v1 --openai-mode completions
```

If workout count is not auto-detected, specify explicitly:
```bash
python -m garmin_fit.llm.request_cli --api openai --url http://127.0.0.1:1234/v1 --workouts 48
```

Or write the YAML manually using `docs/YAML_GUIDE.md` as a reference.

**4.** Build FIT files:
```bash
python -m garmin_fit.cli run
```

**5.** Copy to your Garmin watch:

Files will appear in `Output_fit/`.

- Copy `.fit` files to the **`/GARMIN/New files`** folder on the watch
- The watch will process them automatically — workouts will appear in **`/GARMIN/Workouts`**

A wrapper script is also available:
```bash
run_pipeline.bat   # Windows
./run_pipeline.sh  # Linux / macOS
```

## Commands

### Primary CLI

```bash
python -m garmin_fit.cli run                          # Full workflow
python -m garmin_fit.cli validate-yaml --plan Plan/plan.yaml
python -m garmin_fit.cli validate-fit
python -m garmin_fit.cli doctor
python -m garmin_fit.cli doctor --llm --api openai --url http://127.0.0.1:1234/v1
python -m garmin_fit.cli archive
python -m garmin_fit.cli list-archives
python -m garmin_fit.cli restore <name>
```

### Garmin Calendar Upload

See [Scenario 3](#scenario-3--garmin-connect-calendar-no-usb) for full details and all flags.

```bash
python -m garmin_fit.cli garmin-calendar --plan Plan/plan.yaml --dry-run
python -m garmin_fit.cli garmin-calendar --plan Plan/plan.yaml --year 2026
python -m garmin_fit.cli garmin-calendar --plan Plan/plan.yaml --from-date 2026-06-01 --to-date 2026-06-30
python -m garmin_fit.cli garmin-calendar --plan Plan/plan.yaml --skip-past --year 2026
```

### Garmin Calendar Delete

```bash
python -m garmin_fit.cli garmin-calendar-delete --email you@example.com --password yourpassword --year 2026 --from-date 2026-06-01 --to-date 2026-06-30 --dry-run
python -m garmin_fit.cli garmin-calendar-delete --email you@example.com --password yourpassword --year 2026 --from-date 2026-06-01 --to-date 2026-06-30 --confirm
```

### LLM Generation

```bash
python -m garmin_fit.llm.request_cli --api openai --url http://127.0.0.1:1234/v1
python -m garmin_fit.llm.request_cli --api ollama
python -m garmin_fit.llm.request_cli --workouts 48   # Explicit count
```

### Legacy / Debug CLI

```bash
python -m garmin_fit.legacy_cli templates --plan Plan/plan.yaml
python -m garmin_fit.legacy_cli build
python -m garmin_fit.legacy_cli compare --plan Plan/plan.yaml
```

### Other

```bash
python -m garmin_fit.runner          # Interactive menu (recommended for local LLM users)
python -m garmin_fit.bot             # Telegram bot
python -m garmin_fit.cli validate-yaml --plan Plan/plan.yaml  # Quick validation
```

The interactive menu (`garmin_fit.runner`) covers the full local workflow in one place:
LLM generation → build → validate → Garmin Calendar upload/delete → archive.
